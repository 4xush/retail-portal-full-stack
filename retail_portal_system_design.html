Retail Portal - Full Stack Design (Demo Slides)
Simple reference for architecture, flows, schema, API overview, tech stack, and feature decisions.

Architecture

Design style: Modular monolith. Fast to build, easy to explain in demo, and later we can split modules if scale increases.

Frontend
React SPA
  • UI pages for customer + admin
  • React Router for page flow
  • Zustand for app state
Backend
Node + Express API
  • Auth, products, cart, orders
  • Role checks for admin routes
  • Clear service-based modules
Database
MongoDB
  • Document model for product flexibility
  • Good fit for add-ons and combos
  • Easy catalog iteration in demo phase

Simple index plan

  • users.email unique index
  • products on categoryId and title text search
  • orders on userId + createdAt
  • carts on userId (one active cart per user)

API Design (Overview Only)

Style: REST API with JSON responses.

Base path: /api/v1

ModuleKey endpointsGoal
AuthPOST /auth/signup, POST /auth/loginUser access and role setup
CatalogGET /categories, GET /productsList and search products
AdminPOST /products, PUT /products/:idManage catalog data
CartGET /cart, POST /cart/itemsCart operations
OrdersPOST /orders, GET /ordersPlace and view orders

For demo, keep response shape simple: success, data, message, error.

Tech Stack

Frontend
React + Vite + Tailwind
  • Fast UI development
  • Simple routing and reusable components
State management
Yes, use Zustand
  • Good for auth/cart/global UI state
  • Small and easy compared to Redux setup
Backend
Node.js + Express
  • Simple REST API and middleware flow
  • Same language in full stack
Database
MongoDB in Docker
  • Run local Mongo using Docker container
  • Easy local reset and demo consistency

Feature Solutions

Feature: Login and role-based pages

  • Use: JWT + role field in user document
  • Why this: Stateless auth with clear role control and scalable API access

Feature: Product search and filters

  • Use: Fuse.js fuzzy search on product list + category filter
  • Why this: Fast typo-tolerant search with simple setup and smooth UI response

Feature: Cart with add-ons and combos

  • Use: Cart items saved in DB with selected options
  • Why this: User can continue cart across refresh/login

Feature: Order history accuracy

  • Use: Save item snapshot in order
  • Why this: Past order remains correct even if product changes later

Feature: Fast frontend state

  • Use: Zustand for client state, API state kept in backend responses
  • Why this: Minimal boilerplate and clean stores
